Instructions for building PILS with Juce bindings on MS Windows

Reqired downloads

  • VC 2008 Express if you don't have VC 2008
  • A Subversion client, such as TortoiseSVN
  • The Juce trunk, SVN url: https://juce.svn.sourceforge.net/svnroot/juce/trunk
    (JucePILS will be based on the next version of Juce (presumably 1.47) when that is released. It doesn't work with Juce 1.46
  • The PILS trunk, SVN url: https://juce.svn.sourceforge.net/svnroot/pilsprogramming/trunk

Optional download

  • MySQL 5.1 or later (get the .msi installer, and do a full install) for compiling the MySQL pilsplug.

Compiling Juce

  • Open juce\build\win32\vc8\JUCE.vcproj with Visual Studio. It will be converted to vc9.
  • Unless you have the relevant SDKs installed, outcomment these two lines from juce_Config.h
        #define JUCE_ASIO 1
        #define JUCE_QUICKTIME 1
  • Build the Debug and Release configurations.

Compiling and running PILS

  • Set these environment variables:
    variablevalueexample
    JUCEroot of Juce trunkC:\juce
    MYSQL (optional)MySQL server installation dirC:\Program files\MySQL\MySQL Server 5.1
  • If VS 2008 is running, close it to ensure the environment variables take effect.
  • Open build\win32\vc9\jpils.sln, and compile the Debug and Release versions.
    If you did not install MySQL and set the MYSQL environment variable, the mysql-plug project will fail. Just ignore the failure or exclude the project from the builds.
  • Run the Release configuration. Click Yes to register the .pils filetype.
  • Close everything and open src\scripts\pils\html-generator.pils, to generate the HTML documentation from the openoffice writer files.

Updating the Juce bindings

If you want to update the Juce bindings to incorporate recently added Juce classes and methods, you can try this:
  • Close all PILS programs
  • Back up all jpils-auto*.* files in the src/jpils dir.
  • Run pils\english\system\juce\plumming-generator.pils
  • This will parse the juce-amalgamated.h file in the root of the Juce trunk, and generate updated versions of the auto-files.
  • Try compiling and running the Debug configuration. Don't compile the Release configuration unless the Debug configuration worked - or you will trash the executable needed to run the plumming generator.
